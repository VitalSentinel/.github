# VitalSentinel

**Your website's revenue insurance.**

The worst website problems are the quiet ones. A robots.txt edit takes you out of Google. A deploy ships slower code and Core Web Vitals slide with it. A key page starts 500-ing and no one notices for a week. By the time it shows up in your traffic, the bill is already paid.

VitalSentinel keeps watch and tells you when something breaks, before the damage compounds.

---

## Modules

### Monitoring

| Module | What it catches |
|---|---|
| **RUM Monitoring** | Core Web Vitals regressions in real user data |
| **CrUX Monitoring** | Field-data trends that move before search rankings do |
| **Synthetic Monitoring** | Performance regressions, via scheduled Lighthouse runs |
| **Uptime Monitoring** | Outages and availability issues, around the clock |
| **Robots.txt Monitoring** | Accidental de-indexing the moment the file changes |
| **Indexing Monitoring** | Pages quietly dropping out of Google's index |

### Analytics

| Module | What it shows |
|---|---|
| **Web Analytics** | Privacy-first traffic and anomaly detection |
| **Google Search Console** | Impressions, clicks, queries, and indexing signals |
| **Google Analytics** | GA4 traffic and engagement, side by side with the rest |

Alerts go out by email and in-app the moment a threshold trips.
